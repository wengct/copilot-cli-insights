⚙️ GitHub Copilot CLI 前置設定與啟用教學

本 Dashboard 主要是解析並呈現 GitHub Copilot CLI 的 Status Line (狀態列) 所收集的 Token 數據。我們將使用 ~/.copilot/statusline-token.sh 進行每日數據統計與會話紀錄。

1️⃣ 1. 確認 script 有執行權限

首先建立設定目錄，並將專案中的收集腳本複製至家目錄的 .copilot 目錄下，最後賦予執行權限：

mkdir -p ~/.copilot && cp shell/statusline-token.sh ~/.copilot/statusline-token.sh && chmod +x ~/.copilot/statusline-token.sh

2️⃣ 2. 編輯設定檔

編輯或新增 Copilot CLI 的設定檔 ~/.copilot/settings.json：

nano ~/.copilot/settings.json

在設定檔中加入以下 statusLine 設定內容：

{
  "statusLine": {
    "type": "command",
    "command": "/home//.copilot/statusline-token.sh",
    "padding": 1
  }
}
💡 Home 目錄路徑提示：
如果您的 $HOME 家目錄不是 /home/<username>，可在終端機執行 echo $HOME 查詢您的家目錄路徑，並對應修改 command 欄位的值。

3️⃣ 3. 已經有其他設定時（不要覆蓋）

若您的 settings.json 中已經有其他現成設定，請勿整檔覆蓋，只需將 statusLine 屬性合併加入即可，例如：

{
  "footer": {
    "showDirectory": true,
    "showBranch": true
  },
  "statusLine": {
    "type": "command",
    "command": "/home//.copilot/statusline-token.sh",
    "padding": 1
  }
}

4️⃣ 4. 重開 Copilot CLI

設定完成並存檔後，請退出目前的 Copilot CLI 聊天會話，並重新進入以套用全新設定。

5️⃣ 5. 檢查是否成功

進入 Copilot CLI 會話聊天後，畫面底部應該會看到由本專案腳本收集並精緻渲染出的狀態列，如：

🤖 Auto → GPT-5.3-Codex • #1 • ↑ 22.8k • c 22.0k/0 • ↓ 61 • r 29 • total 22.9k • +22.9k • last 22.8k/61 • ctx 9%
⚠️ 除錯與檢查 (Troubleshooting)：

🔍 A. 若狀態列未正常出現，請先單獨測試腳本是否能正常執行：

echo '{}' | ~/.copilot/statusline-token.sh

🔍 B. 請確認 settings.json 是合法的 JSON 格式：

jq . ~/.copilot/settings.json
